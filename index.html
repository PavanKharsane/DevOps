1. Linux — Foundations

Key topics: CLI, filesystem, permissions, processes, networking, package management, shell scripting

Concepts: kernel vs userland, distributions (Ubuntu, CentOS, Amazon Linux, Debian), shell (bash, sh, zsh).

Important commands:

pwd            # print working directory
ls -la         # list files with details
cd /path       # change directory
cat file.txt   # view file
less file.txt  # paginated view
cp src dest    # copy
mv src dest    # move/rename
rm file        # remove file
mkdir dir      # create directory
rmdir dir      # remove directory (if empty)

Permissions & Ownership:

ls -l          # view permissions and owner
chmod 644 file  # set file permissions
chown user:grp file  # change owner and group

Understand /, /home, /etc, /var, /tmp, /usr — mounting, fstab, partitions, df, du.

df -h           # disk free (human-readable)
du -sh folder    # folder size
mount /dev/xvdf /data   # mount device
ps aux | grep process
top               # interactive process viewer
systemctl status httpd   # service management (systemd)
sudo systemctl start|stop|restart

IP tools: ip, ifconfig (legacy), ss, netstat. SSH basics and key-based auth.

# check ip
ip addr show

# ssh into server
ssh -i /path/key.pem ec2-user@ec2-public-ip

# copy file via scp
scp -i key.pem file.txt ec2-user@host:/home/ec2-user/

Write repeatable tasks in scripts. Basic template:

#!/bin/bash
set -euo pipefail

# Example: backup a folder
src="/home/ubuntu/data"
dest="/backup/$(date +%F).tar.gz"
tar -czf "$dest" "$src"

2. AWS Basics

Core cloud concepts, regions, availability zones, shared responsibility model

Cloud fundamentals: cloud = on-demand compute, storage, networking. Key benefits: elasticity, scalability, pay-as-you-go.

Regions & AZs: AWS divides the world into Regions (e.g., us-east-1) and AZs (e.g., us-east-1a) for high availability.

IAM (Identity & Access Management): principle of least privilege, users, groups, roles, policies, temporary creds (STS).

# example AWS CLI to get caller identity
aws sts get-caller-identity

3. EC2 — Elastic Compute Cloud

Launch, connect, store, and secure virtual servers

Instance families: general purpose (t3/t4), compute-optimized (c5), memory-optimized (r5), storage-optimized. Choose based on workload.

AMI — Amazon Machine Image (e.g., Amazon Linux 2, Ubuntu). AMI contains OS and initial config.

# quick AWS CLI example to launch (simplified)
aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --instance-type t3.micro \
  --key-name my-key \
  --security-group-ids sg-0123456789abcdef0 \
  --subnet-id subnet-0abc12345

# connect (ssh)
ssh -i my-key.pem ec2-user@{PUBLIC_IP}

Security groups: act as virtual firewall. Open only required ports (22 for SSH, 80/443 for web).

EBS: block storage, attach to one instance (unless using multi-attach). Snapshots for backups.

EFS: shared file system (NFS) for multiple instances.

# create snapshot
aws ec2 create-snapshot --volume-id vol-0123456789abcdef0 --description "backup"

VPC isolates your cloud network. Subnets can be public (with Internet Gateway) or private. Use NAT Gateway for outbound internet from private subnets.

4. S3 — Simple Storage Service

Object storage: buckets, objects, versioning, policies, lifecycle, encryption

Core concepts: S3 stores objects in buckets. Objects have keys and metadata. S3 is region-specific and provides strong durability.

# create a bucket (example)
aws s3api create-bucket --bucket my-example-bucket --region us-east-1

# upload file
aws s3 cp file.txt s3://my-example-bucket/folder/file.txt

# list objects
aws s3 ls s3://my-example-bucket --recursive

Enable versioning to protect from accidental deletes. Use bucket policies or IAM to control access; prefer IAM roles over ACLs.

# enable versioning
aws s3api put-bucket-versioning --bucket my-example-bucket --versioning-configuration Status=Enabled

Lifecycle rules move objects to cheaper storage (GLACIER/DEEP_ARCHIVE) or expire them after X days to save cost.

{
  "Rules": [{
    "ID":"MoveToIA",
    "Status":"Enabled",
    "Filter":{ "Prefix":"logs/" },
    "Transitions":[{"Days":30,"StorageClass":"STANDARD_IA"}]
  }]
}

S3 supports server-side encryption (SSE-S3, SSE-KMS), client-side encryption, and bucket policies to enforce HTTPS and encryption.

5. AWS CLI & Automation

Install, configure, and run common commands

# install (linux)
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# configure
aws configure
# Provide AWS Access Key ID, Secret, region, output format

Tips: Use IAM roles for EC2, avoid embedding long-term keys in code, use named profiles for multiple accounts.

6. Practical Labs & Mini Projects

Hands-on tasks to reinforce learning
  • Linux: Build a bash script that rotates logs, compresses them, and moves to /backup.
  • EC2: Launch an EC2 instance, host a simple Nginx site, attach an EBS volume, and snapshot it.
  • S3: Host a static website using S3, set up bucket policy for public read (or CloudFront for secure delivery).
  • Automation: Create a small Terraform script (or CloudFormation) that provisions EC2 + Security Group + S3 bucket.
